# GhostClaim Shield

**AI-powered detection of fraudulent death certificates used to steal pensions and insurance of living elderly.**

Built for **Bharat Academix CodeQuest 2026 — Round 2: Prototype Development**  
**Team IndoriPohe** — Anushka Parihar, Anuj Billore (SGSITS Indore)

---

## Overview

GhostClaim Shield is a full-stack prototype that screens death certificates against pension and Aadhaar records to detect fraud targeting elderly citizens. When a certificate is flagged, the system explains *why* it was scored, alerts the elderly person (simulated IVR), and logs every action to a tamper-proof audit chain.

**Tech stack:** Node.js + Express · React + Vite + Tailwind · JSON file storage (no DB setup required)

## Quick start

Clone the repo, then run backend and frontend in two terminals.

**Terminal 1 — backend**
```bash
cd backend
npm install
npm run seed    # run once after clone — loads 8 demo cases
npm run dev     # http://localhost:4000
```

**Terminal 2 — frontend**
```bash
cd frontend
npm install
npm run dev     # http://localhost:5173
```

Open **http://localhost:5173** → sign in with any officer name → dashboard loads with 8 seeded cases (4 HIGH risk, 4 LOW).

> Demo data is generated by `npm run seed` and stored in `backend/src/data/` (gitignored).

## Project structure

```
GhostClaim/
├── backend/          Express API — risk engine, alerts, audit trail
├── frontend/         React dashboard — officer UI
└── docs/             Technical documentation
```

The backend persists to JSON files via `backend/src/db/jsonStore.js`. No PostgreSQL or external services needed — swap in a real database later by changing only that file.

## Features

| Screen | What it does |
|---|---|
| **Officer Sign In** | Demo landing screen before dashboard |
| **Dashboard** | Stats, district fraud hotspots, case list with search + risk filter |
| **Case Detail** | TrustMeter score, explainable signal breakdown, guardian alerts, investigation actions |

| **Print Case Report** | Clean printable government-style case report |

## Four core components

| # | Component | Implementation |
|---|---|---|
| A | **AI Anomaly Detection** | Weighted multi-signal scoring engine — 7 fraud signals, 0–100 score, HIGH/MEDIUM/LOW band ([`riskEngine.js`](backend/src/services/riskEngine.js)) |
| B | **Cross-Database Verification** | Registrar form collects EPFO/LIC/Aadhaar cross-check fields (simulated as form inputs — no public API sandbox exists) |
| C | **Guardian Alert System** | Auto-triggers on HIGH risk; multilingual IVR messages in Hindi, Marathi, Tamil, Bengali, English ([`alertService.js`](backend/src/services/alertService.js)) |
| D | **Audit Trail** | Cryptographic SHA-256 hash chain — tamper-evident, independently verifiable ([`auditTrail.js`](backend/src/services/auditTrail.js)) |

## Demo walkthrough (~3 min)

1. **Sign in** with an officer name on the landing screen
2. **Dashboard** — stats, district hotspots, search `"Kamla"` or filter **HIGH**
3. **Case Detail** — open a HIGH-risk case, show TrustMeter + signal breakdown
4. **Guardian Alert** — click **Press 1 — I'm Alive**, show toast + investigation hold
5. **Register Certificate** — submit a suspicious certificate (different districts, no hospital record), watch live scoring
6. **Audit Trail** — show hash chain, click **Re-verify**

## What's simulated vs. real

| Area | Prototype | Production would use |
|---|---|---|
| Fraud scoring | Explainable rules engine | Trained ML model (XGBoost) on labeled fraud data |
| Guardian alerts | Simulated IVR/SMS with response buttons | Live Twilio / AWS Connect calls |
| Cross-checks | Form inputs | EPFO / UIDAI / LIC API integrations |
| Audit trail | SHA-256 hash chain in JSON | Same logic on blockchain or immutable DB |
| Officer login | Demo gate (name in memory) | JWT + role-based authentication |

> **For judges:** The scoring logic, alert lifecycle, and audit chain are fully working. External integrations are mocked because no public sandboxes exist for EPFO/UIDAI and live phone calls are fragile for demos. The rules engine is intentionally explainable — officers see *why* a case was flagged, not just a black-box score.
